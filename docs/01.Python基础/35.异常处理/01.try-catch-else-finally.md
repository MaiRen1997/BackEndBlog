## try except

### 基础使用

#### 语法

```python
try:
    可能发生异常的代码
except:
    异常处理的代码

```

1. 如果没有发生异常，程序会忽略except中的代码，继续向下执行。
2. 如果发生了异常，会忽略try中剩余代码，执行except中的代码。

#### 案例

```python
try:
    result = 3 / 1
    print("没有发生异常")
except:
    print("发生异常了")
print("End")

```

### 捕获不同异常

1. 取别名
2. 处理不同异常
3. 多个异常合并处理
4. 最后的默认异常处理

#### 语法

```python
try:
    可能发生异常的代码
except 异常类型1 as 变量名1:
    异常处理的代码
except 异常类型2 as 变量名2:
异常处理的代码
except(异常类型3, 异常类型4, 异常类型5) as 变量名3:
异常处理的代码
except:
异常处理的代码
```

1. 如果没有发生异常，程序会忽略except中的代码，继续向下执行。
2. 如果发生了异常，会忽略try中剩余代码，根据异常类型匹配到相应的 except 并执行其中的代码。
3. 如果发生了异常，且异常类型无法和任何except匹配，异常将向外传递。
4. 一个except可以同时处理多个异常，将这些异常放在一个元组中。
5. 最后一个 except 可以忽略异常类型，它将被作为通配符使用，`除上面的列举的异常，都会走这里`

#### 案例

```python
try:
    result = 3 / 0
print("发生异常了")
except ZeroDivisionError as e:
    print(e)
except (RuntimeError, TypeError, NameError) as e:
    print(e)
except:
    print("Unexpected error")
print("End")
```

## else的使用

可选地将else放在所有except之后。如果try中代码没有发生异常，将执行 else 中的代码。

### 语法

```python
try:
    可能发生异常的代码
except 异常类型1 as 变量名1:
    异常处理的代码
except 异常类型2 as 变量名2:
    异常处理的代码
else:
    没有异常时执行的代码
```

### 说明 

1. `从执行效果上说，将代码放到else块和直接放到try块中是一样的。`
2. 将try正常执行完毕而没有引发任何异常后被执行的代码放到else中。提供了一种清晰的逻辑区分，将正常情况的代码与异常处理代码分开，使代码更易于理解和维护，有助于代码的可读性和可维护性。
3. 例如，你希望在try块中有些操作执行成功后，再执行其它代码，那就可以把代码放到else语句块中。

### 案例

```python
try:
    result = x / y
except ZeroDivisionError:
    print("除数不能为零！")
else:
    print(f"结果是: {result}")
```

## finally的使用

可选地，放在最后。无论是否发生异常都会执行的代码，通常用于执行一些必须要进行的清理操作，例如关闭文件、释放资源（如网络连接、数据库连接、锁等），即使在执行 try 块中的代码时出现了异常，也能保证这些操作得以完成

### 语法

```python
try:
    可能发生异常的代码
except 异常类型1 as 变量名1:
    异常处理的代码
except 异常类型2 as 变量名2:
    异常处理的代码
else:
    没有异常时执行的代码
finally:
    无论是否发生异常都会执行的代码
print('end')
```

### 说明

1. 如果从执行效果上说，大部分场景，将代码放到finally语句和放到try-except块外效果是一样的。
2. **但是，如果发生的异常是`除异常类型1、异常类型2`之外的异常，说明没有被捕获，最后的print就不会被执行，此时，执行效果是不同的**
3. finally 语句块确保了无论 try 块中是否发生异常，也无论 except 块是否被执行，其中的代码都会被执行
4. 直接放在try-except结构外面的代码只会在try-except结构正常执行完毕后才会执行，

   如果在try块中出现异常且没有被except块捕获，或者在except块中出现了新的异常导致程序终止，

   那么这部分代码将**不会被执行**

5. 一般在finally中，进行资源的释放，以及一些连接的操作

### 案例

```python
# try:
#     result = 3 / 0
# except ZeroDivisionError as e:
#     print(e)
# else:
#     print(result)
# finally:
#     print("finally")
# print("End")
#输出结果：
# division by zero
# finally
# End
try:
    result = 3 / 0
except NameError as e:
    print(e)
else:
    print(result)
finally:
    print("finally")
print("End")
#输出结果：

#   File "e:\Hello\hello.py", line 15, in <module>

#              ~~^~~

```
