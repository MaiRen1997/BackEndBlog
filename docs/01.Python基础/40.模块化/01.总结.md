## 不带包的模块导入

### 全局导入

​        语法：
​            import 模块名 [as 别名]
​        访问：
​            模块名.成员名

### 局部导入

​        语法：
​            from 模块名 import 成员名1[as 别名], 成员名2[as 别名],…
​        说明：
​            指定导入模块的部分成员，只能使用其导入的成员,未导入的成员不能使用。
​            如果多个模块中存在重名成员，后一次导入会覆盖前一次导入。
​        访问：
​            直接通过成员名的方式访问

### 局部导入

​        语法：
​            from 模块名 import *
​        说明：
​            导入模块中所有不以单下划线开头的成员
​        访问：
​            通过成员名的方式访问

## __all属性

​    前提：在使用from 模块名 import *的时候
​    说明：在被导入的模块中，通过__all__属性声明，哪些成员可以被导入
​         就算是当前成员是以_开头，如果声明在__all__中，也可以被外部访问

4.__name__属性
    __name__ 是一个特殊的内置变量
    当一个Python文件被直接运行时，该文件的__name__属性值为"__main__"。
    当一个Python文件作为模块被导入时，__name__属性会被设置为该模块的名称（即文件名，不包含 .py 后缀）
    如果在导入其它模块的时候，不需要被导入模块的测试代码被执行，那么可以在被导入模块中加如下分支判断
        if __name__="__main__":
            执行测试语句

5.dir()函数
    dir() 是一个内置函数，主要用于列出对象的属性和方法，或者列出当前作用域中定义的名称，并以一个字符串列表的形式返回。

6.带包的模块导入
    全局导入
        语法：
            import 包名.模块名 [as 别名]
        说明：
            导入包中模块的所有成员
            如果import导入的时候，最后一项是包，python会进行限制，不能将包下的所有模块的所有成员都导入
            需要在被导入的包的__init__.py文件中，加入导入哪些模块的限制。
            一般在__init__.py中使用相对路径的导入方式

        访问:
            包名.模块名.成员名或者别名.成员名
    
    局部导入
        语法：
            from 包名 import 模块名 [as 别名]
        说明：
            导入包中的某一个模块的所有成员
        访问：
            模块名.成员名
    局部导入
        语法：
            from 包名.模块名 import 成员名 [as 别名]
        说明：
            导入包中的某一个模块的某一个成员
        访问：
            成员名
    
    局部导入
        语法：
            from 包名.模块名 import *
        说明：
            导入包中的某一个模块中不以下划线开始的成员
        访问：
            成员名
    
    局部导入
        语法：
            from 包名 import *
        说明：
            导入包中的所有模块
        访问：
            模块名.成员名
        注意：
            必须在包的__init__.py中通过__all__指定哪些模块可以被导入


7.相关概念对比
    遍历:将数据一个个取出
    循环:重复执行一段代码,是实现遍历的一种方式
    迭代:也是实现遍历的一种方式，底层是迭代器

    for循环底层其实就是迭代器
    
    Iterable:可迭代对象
        能直接使用for循环进行遍历  底层实现了__iter__,当调用iter()方法的时候会被调用
    
    Iterator:迭代器
        底层实现了__iter__,当调用iter()方法的时候会被调用
        底层实现了__next__,当调用next()方法的时候会被调用
