## yield的使用

使用 yield 是为了**实现“先执行一部分代码，暂停，然后回来执行剩下的代码”这一流程**，而 **return 会直接终止函数**，无法实现这个流程

```python
# 生成器函数 - 逐个产生值
def get_numbers_yield():
    for i in range(5):
        yield i  # 每次产生一个值，然后暂停

gen = get_numbers_yield()
print(gen)  # <generator object get_numbers_yield at 0x...>
print(next(gen))  # 0
print(next(gen))  # 1
print(next(gen))  # 2
print(list(gen))  # [3, 4] - 剩下的值
```

## 创建数据库会话

### 创建同步会话

#### 定义session

```python
from sqlmodel import Session
from contextlib import contextmanager, asynccontextmanager
from typing import Iterator, AsyncIterator
import logging

@contextmanager
def session_getter() -> Iterator[Session]:
    session = Session(engine)

    try:
        yield session
    except Exception as e:
        logger.info('Session rollback because of exception:{}', e)
        session.rollback()
        raise
    finally:
        session.close()
```

#### 使用session

1. 使用时，要用with

   ```python
   async def create_agent( cls,  agent: AgentTable ):
           with session_getter() as session:
               session.add(agent)
               session.commit()
               session.refresh(agent)
               return agent
   ```

### 创建异步会话

#### 定义session

```python
from sqlmodel.ext.asyncio.session import AsyncSession
from contextlib import contextmanager, asynccontextmanager
from typing import Iterator, AsyncIterator
import logging

@asynccontextmanager
async def async_session_getter() -> AsyncIterator[AsyncSession]:
    session = AsyncSession(async_engine)  # 使用异步引擎创建会话

    try:
        yield session
    except Exception as e:
        logger.info('Session rollback because of exception: %s', e)
        await session.rollback()  # 异步回滚
        raise
    finally:
        await session.close()  # 异步关闭会话
```

#### 使用session

1. 使用时，要用 async with

   ```python
   async def create_agent_skill(cls, agent_skill: AgentSkill):
           async with async_session_getter() as session:
               session.add(agent_skill)
               await session.commit()
               await session.refresh(agent_skill)
               return agent_skill
   ```

### 为什么要用@contextmanager

| 对比项           | 不加` @contextmanager`                                       | 不加 `@contextmanager`                                       |
| ---------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **函数返回类型** | 一个 **生成器对象** (Generator)                              | 一个 **上下文管理器** (Context Manager)                      |
| **with 兼容性**  | **不兼容** ，会直接报错                                      | **完全兼容**                                                 |
| **核心作用**     | 只能作为迭代器使用（如` for `循环或 `next()` ）              | 专门用于配合 `with` 语句，实现资源的自动获取和释放           |
| **带来的影响**   | **无法实现** 预期的资源管理自动化，破坏了代码设计的初衷，导致调用方代码出错。 | **实现了** 简洁、安全、可复用的资源管理模式，是编写健壮代码的最佳实践 |

#### 总结

1. `with`只是一个语法糖，它在工作时，只认识一种东西： **上下文管理器协议**（Context Manager Protocol） 。
2. 上下文管理协议：这个协议非常简单，一个对象只要实现了以下两个方法，就可以配合 with 语句使用：
   1. `__enter__(self)` : 在进入 with 块时被调用，它的返回值会被赋给 as 后面的变量。
   2. `__exit__(self, exc_type, exc_value, traceback)` : 在离开 with 块时被调用，负责清理工作。如果 with 块中发生了异常，异常的类型、值和追溯信息会作为参数传给它。
3. `@contextmanager` 装饰器的唯一作用，就是充当一个“翻译官” 。它能把你用 yield 写的、非常易读的 生成器函数 ，“翻译”**成一个遵循上述协议的、带有` __enter__ `和` __exit__ `方法的 标准上下文管理器对象**

### 定义session 为什么要用yield

1. 如果将yield 换成return ,虽然session被返回出去了，但是except和finally都不会执行
2. 而使用yield，当代码运行到yield，
   会将session交给with所在的函数create_agent_skill
   create_agent_skill函数执行完毕后，将在async_session_getter函数内部，继续执行
   如果错误，执行except，如果正确，执行finally

## session会话操作

| 分类               | 方法名               | 作用简述                                                     | 使用示例 / 备注                                              |
| :----------------- | :------------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| **核心操作**       | `add(instance)`      | 将一个新对象加入到会话中，等待插入。                         | `session.add(hero)`                                          |
|                    | `add_all(instances)` | 批量将多个对象加入会话。                                     | `session.add_all([hero1, hero2])`                            |
|                    | `commit()`           | 提交当前事务，将所有待处理的更改（增、删、改）永久保存到数据库。 | `session.commit()`                                           |
|                    | `refresh(instance)`  | **这是你常用的方法**。从数据库重新加载该对象的最新状态，常用于获取自增 ID。 | `session.refresh(hero)`                                      |
|                    | `delete(instance)`   | 标记一个对象待删除，执行 `commit()` 后生效。                 | `session.delete(hero)`                                       |
| **查询数据**       | `exec(statement)`    | **这是 SQLModel 推荐的核心查询方法**。执行一个 `select` 语句，返回结果。 | `results = session.exec(select(Hero).where(Hero.name == "Spider-Boy"))` |
|                    | `get(entity, ident)` | 通过主键快速获取一个对象，这是最精准的查询方式。             | `hero = session.get(Hero, 1)`                                |
| **事务与生命周期** | `rollback()`         | 回滚当前事务，撤销自上次 `commit()` 以来的所有更改。         | `session.rollback()`                                         |
|                    | `flush()`            | 将当前所有待处理的SQL语句发送到数据库执行，但**不提交事务**。常用于提前获取数据库生成的ID。 | `session.flush()` 之后可以立即访问 `hero.id`，但数据还未真正固化，可被回滚。 |
|                    | `expunge(instance)`  | 从会话中移除一个对象，使其变为"游离态"，之后会话将不再跟踪该对象的任何更改。 | `session.expunge(hero)`                                      |
|                    | `close()`            | 关闭会话，并回滚所有未提交的事务，释放连接资源。通常用 `with Session(engine) as session:` 语句自动管理。 | `session.close()`                                            |

### 属性区分

#### **`flush()` vs `commit()`**：

- **`flush()`** 是把命令发送给数据库执行（此时会生成自增ID等），但数据还在一个**未提交的事务**中。如果后面执行了 `rollback()`，这些数据会被撤销。它主要用于在提交前获取一些数据库生成的值。
- **`commit()`** 是在 `flush()` 的基础上，执行了数据库的 `COMMIT` 命令，将数据**永久保存**。一个完整的保存流程通常是 `add` -> `flush` (可选) -> `commit` -> `refresh` (可选)。

#### **`refresh()` 的作用**

在你提交 (`commit()`) 之后，你添加的对象虽然已经存入了数据库，但其本地的属性（尤其是数据库自动生成的 `id` 或默认时间戳）可能还未更新。`refresh()` 会执行一次新的查询，用数据库里的真实值来更新你的Python对象，这就是为什么你需要在 `commit()` 后调用它

```python
async def create_agent(
    cls,
    agent: AgentTable
):
    with session_getter() as session:
        session.add(agent)
        session.commit()
        session.refresh(agent)
        return agent
```

#### **`get()` vs `exec(select(...))`**

- `session.get(Hero, 1)` 非常简洁，专为"通过主键取一条"这个场景设计，代码意图很清晰。
- `session.exec(select(Hero).where(Hero.id == 1))` 功能更强大，适用于所有复杂的查询条件，并可以灵活控制返回结果的数量（如 `.first()`, `.all()`）

### 示例

#### **add() - 添加单个对象**

```python
from sqlmodel import SQLModel, Field, create_engine, Session

class Hero(SQLModel, table=True):
    id: int | None = Field(default=None, primary_key=True)
    name: str
    age: int | None = None

# 创建引擎和表
engine = create_engine("sqlite:///database.db")
SQLModel.metadata.create_all(engine)

with Session(engine) as session:
    # 添加单个对象
    hero = Hero(name="Superman", age=30)
    session.add(hero)
    session.commit()
    print(f"添加成功，ID: {hero.id}")  # 添加成功，ID: 1
```

####  **add_all() - 批量添加**

```python
with Session(engine) as session:
    # 批量添加多个对象
    heroes = [
        Hero(name="Batman", age=35),
        Hero(name="Wonder Woman", age=28),
        Hero(name="Flash", age=25)
    ]
    session.add_all(heroes)
    session.commit()
    
    for hero in heroes:
        print(f"添加成功: {hero.name}, ID: {hero.id}")
    # 添加成功: Batman, ID: 2
    # 添加成功: Wonder Woman, ID: 3
    # 添加成功: Flash, ID: 4
```

#### **commit() - 提交事务**

```python
with Session(engine) as session:
    hero = Hero(name="Aquaman", age=32)
    session.add(hero)
    
    # 此时数据还未真正写入数据库
    print("添加后但未提交，数据库中没有这条记录")
    
    # commit 将数据永久保存到数据库
    session.commit()
    print("提交后，数据已永久保存")
    print(f"英雄ID: {hero.id}")  # 现在可以获取ID了
```

#### **refresh() - 刷新对象**

```python
with Session(engine) as session:
    # 假设数据库有 server_default 默认值
    class Product(SQLModel, table=True):
        id: int | None = Field(default=None, primary_key=True)
        name: str
        created_at: str = Field(server_default="CURRENT_TIMESTAMP")
    
    product = Product(name="Laptop")
    session.add(product)
    session.commit()
    
    # commit 后，product.created_at 仍然是 None
    print(f"刷新前: {product.created_at}")  # None
    
    # refresh 从数据库重新加载
    session.refresh(product)
    print(f"刷新后: {product.created_at}")  # 2024-01-15 10:30:00
```

#### **delete() - 删除对象**

```python
with Session(engine) as session:
    # 先查询要删除的英雄
    hero = session.exec(select(Hero).where(Hero.name == "Flash")).first()
    
    if hero:
        print(f"准备删除: {hero.name}")
        session.delete(hero)  # 标记删除
        session.commit()  # 执行删除
        print("删除成功")
        
        # 验证是否删除成功
        deleted = session.get(Hero, hero.id)
        print(f"验证: {deleted}")  # None
```

#### **exec() - 执行查询（核心查询方法）**

```python
from sqlmodel import select

with Session(engine) as session:
    # 查询所有
    statement = select(Hero)
    results = session.exec(statement)
    all_heroes = results.all()
    print(f"所有英雄: {[h.name for h in all_heroes]}")
    
    # 条件查询
    statement = select(Hero).where(Hero.age > 30)
    results = session.exec(statement)
    older_heroes = results.all()
    print(f"年龄大于30: {[h.name for h in older_heroes]}")
    
    # 查询第一条
    statement = select(Hero).where(Hero.name == "Batman")
    hero = session.exec(statement).first()
    print(f"找到英雄: {hero.name if hero else 'None'}")
    
    # 查询单个（确保只有一个）
    statement = select(Hero).where(Hero.name == "Superman")
    hero = session.exec(statement).one()
    print(f"唯一英雄: {hero.name}")
```

#### **get() - 通过主键获取**

```python
with Session(engine) as session:
    # 通过 ID 直接获取
    hero = session.get(Hero, 1)
    print(f"ID为1的英雄: {hero.name if hero else '不存在'}")
    
    # 获取不存在的 ID
    hero = session.get(Hero, 999)
    print(f"ID为999: {hero}")  # None
    
    # 批量获取多个
    ids = [1, 2, 3]
    for id in ids:
        hero = session.get(Hero, id)
        if hero:
            print(f"找到: {hero.name}")
```

#### **rollback() - 回滚事务**

```python
with Session(engine) as session:
    try:
        # 添加一个英雄
        hero1 = Hero(name="Test Hero 1", age=20)
        session.add(hero1)
        
        # 故意制造错误（比如添加重复数据）
        hero2 = Hero(name="Test Hero 2", age=25)
        session.add(hero2)
        
        # 假设这里违反了唯一约束
        # 会抛出异常
        
        session.commit()  # 如果失败，不会执行到这里
    except Exception as e:
        print(f"发生错误: {e}")
        session.rollback()  # 回滚所有操作
        print("已回滚，hero1 和 hero2 都未保存")
        
        # 验证：查询不到任何 Test Hero
        statement = select(Hero).where(Hero.name.contains("Test"))
        results = session.exec(statement).all()
        print(f"测试英雄数量: {len(results)}")  # 0
```

#### **flush() - 刷新但不提交**

```python
with Session(engine) as session:
    # 添加新英雄
    hero = Hero(name="Flush Test Hero", age=100)
    session.add(hero)
    
    # flush 会发送到数据库，但不提交
    session.flush()
    
    # flush 后，ID 已经生成（如果是自增）
    print(f"flush 后的 ID: {hero.id}")  # 有值了！
    
    # 此时数据还在事务中，可以回滚
    session.rollback()
    print("回滚后，这个英雄不会被保存")
    
    # 验证
    session.commit()  # 但此时已经回滚，commit 不会有任何效果
```

#### **expunge() - 从会话中移除**

```python
with Session(engine) as session:
    hero = session.get(Hero, 1)
    
    # hero 现在被 session 跟踪
    print(f"hero 在 session 中: {hero in session}")  # True
    
    # 修改属性
    hero.age = 99
    
    # 从 session 中移除
    session.expunge(hero)
    print(f"移除后还在 session 中: {hero in session}")  # False
    
    # 修改不会自动同步到数据库
    session.commit()  # 这个修改不会保存
    
    # 需要重新添加才能保存
    session.add(hero)
    session.commit()  # 现在会保存
```

#### **close() - 关闭会话**

```python
# 手动管理会话（不推荐，但演示用）
session = Session(engine)
try:
    hero = Hero(name="Manual Hero", age=50)
    session.add(hero)
    session.commit()
finally:
    session.close()  # 手动关闭
    print("会话已关闭")
    
    # 关闭后不能再使用
    try:
        session.get(Hero, 1)  # 会抛出异常
    except Exception as e:
        print(f"关闭后使用会出错: {e}")

# 推荐使用上下文管理器，自动关闭
with Session(engine) as session:
    hero = Hero(name="Auto Hero", age=60)
    session.add(hero)
    session.commit()
# 退出 with 块时自动调用 close()
```

#### **merge() - 合并对象**

```python
with Session(engine) as session:
    # 从数据库获取一个对象
    hero = session.get(Hero, 1)
    print(f"原始年龄: {hero.age}")  # 假设是 30
    
    # 关闭 session
    session.close()

# 新 session
with Session(engine) as session:
    # hero 现在是"游离态"对象（不属于任何 session）
    hero.age = 35  # 修改
    
    # merge 将对象合并到新 session
    merged_hero = session.merge(hero)
    print(f"合并后的年龄: {merged_hero.age}")  # 35
    
    session.commit()  # 保存更改
```
