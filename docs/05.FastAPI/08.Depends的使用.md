## 常用场景

| **分页参数**   | 提取 `skip`、`limit`、`page` 等公共参数 |
| -------------- | --------------------------------------- |
| **用户认证**   | 验证 Token/API Key，返回用户信息        |
| **数据库会话** | 获取数据库连接/会话，请求结束后自动关闭 |
| **权限校验**   | 检查用户角色或权限                      |
| **请求日志**   | 记录请求信息，不关心返回值              |
| **缓存处理**   | 检查缓存，决定是否从数据库查询          |

## Depends与中间件的区别

| 特性               | `Depends`          | 中间件                 |
| :----------------- | :----------------- | :--------------------- |
| **作用粒度**       | 单个或一组特定路由 | 全局所有请求           |
| **控制精度**       | 精确到每个接口     | 只能整体控制           |
| **访问路径参数**   | ✅ 可以             | ❌ 不能（只有原始请求） |
| **访问接口返回值** | ❌ 不能             | ✅ 可以（响应后处理）   |
| **依赖注入能力**   | ✅ 可嵌套、可复用   | ❌ 无依赖注入           |
| **路由匹配前执行** | ❌ 匹配到路由后     | ✅ 最先执行             |

## Depends的参数解析

### 基础使用

1. 浏览器通过offset、pageSize发送参数
2. Depends将参数原封不动返回
3. 经过Query进行取别名，转换成skip、limit，进行逻辑运算

```python
from fastapi import Query

def query_dependency(
    skip: int = Query(0, alias="offset"),  # 客户端发送 ?offset=10
    limit: int = Query(100, alias="pageSize")  # 客户端发送 ?pageSize=20
):
    return {"skip": skip, "limit": limit}

# 客户端请求: /items?offset=10&pageSize=20
@app.get("/items")
def get_items(params: dict = Depends(query_dependency)):
    return params  # 返回 {"skip": 10, "limit": 20}
```

### 获取request

以上比较简洁的，直接获取了参数offset和pageSize，如果需要获取更多的request信息，可以在形参上添加request

```python
from fastapi import Request, Depends, HTTPException

def query_dependency(
    request: Request, # 随时可以加进来
    skip: int = Query(0, alias="offset"),
    limit: int = Query(100, alias="pageSize")
):
    print(f"请求路径是: {request.url.path}")
    return {"skip": skip, "limit": limit}
```
