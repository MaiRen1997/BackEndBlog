## 什么是中间件

中间件是在**请求到达路由处理函数之前**和**响应返回客户端之前**执行的函数，可以在处理流程中插入自定义逻辑。

## 基本作用

1. **请求预处理**：日志记录、身份验证、请求解析
2. **响应后处理**：添加响应头、压缩响应、错误处理
3. **拦截请求**：权限检查、限流、请求过滤
4. **性能监控**：请求耗时统计
5. 中间件无法获取到路径参数：如`/{user_id}`中的 `user_id`

## 基本语法

```python
from fastapi import FastAPI
from starlette.middleware.base import BaseHTTPMiddleware
from fastapi.middleware.cors import CORSMiddleware
app = FastAPI()

# 方式1：使用内置中间件
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# 方式2：自定义中间件
class CustomMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        # 请求前处理
        print("Before request")
        
        # 调用下一个中间件或路由处理函数
        response = await call_next(request)
        
        # 响应后处理
        print("After request")
        
        return response

app.add_middleware(CustomMiddleware)
```

## 执行顺序

中间件的执行顺序与添加顺序**相反**

```python
from fastapi import FastAPI
from starlette.middleware.base import BaseHTTPMiddleware
import time

class MiddlewareA(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        print("A: Before")
        response = await call_next(request)
        print("A: After")
        return response

class MiddlewareB(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        print("B: Before")
        response = await call_next(request)
        print("B: After")
        return response

app = FastAPI()

# 添加顺序：先A，后B
app.add_middleware(MiddlewareA)
app.add_middleware(MiddlewareB)

@app.get("/")
async def root():
    print("Route handler")
    return {"message": "Hello"}

# 执行顺序：
# 请求 → A:Before → B:Before → Route handler → B:After → A:After → 响应
# 输出：
# A: Before
# B: Before
# Route handler
# B: After
# A: After
```

## 参数分解

### request

`request` 实际上是 **`starlette.requests.Request`** 对象，FastAPI 只是重新导出了它。

#### 主要属性和方法

```python
from fastapi import FastAPI, Request
from starlette.middleware.base import BaseHTTPMiddleware

class RequestInfoMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        # 1. 请求基本信息
        print(f"Method: {request.method}")           # GET, POST等
        print(f"URL: {request.url}")                 # 完整URL
        print(f"Path: {request.url.path}")           # 路径部分
        print(f"Query params: {request.url.query}")  # 查询参数
        
        # 2. 请求头
        headers = request.headers
        user_agent = headers.get("user-agent")
        auth = headers.get("authorization")
        print(f"Headers: {dict(headers)}")
        
        # 3. 客户端信息
        client_host = request.client.host
        client_port = request.client.port
        print(f"Client: {client_host}:{client_port}")
        
        # 4. 请求体（注意：只能读取一次！）
        # 方式1：直接读取
        body = await request.body()
        print(f"Body: {body}")
        
        # 方式2：作为JSON解析
        try:
            json_body = await request.json()
            print(f"JSON: {json_body}")
        except:
            pass
        
        # 5. 路径参数和查询参数
        # 注意：这些通常需要在路由处理函数中获取
        # 在中间件中无法直接获取已解析的参数
        
        # 6. 自定义状态存储
        request.state.user_id = "123"  # 存储数据供后续使用
        request.state.request_time = datetime.now()
        
        response = await call_next(request)
        return response

# 在路由中可以访问
@app.get("/test")
async def test_endpoint(request: Request):
    user_id = request.state.user_id  # 获取中间件存储的数据
    return {"user_id": user_id}
```

### call_next

具体来说，它是一个异步函数：输入：Request 对象，输出：Response 对象的协程

#### 作用

```python
class CallNextDemoMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        # 1. 调用下一个中间件或路由处理函数
        # 这是整个中间件最关键的调用
        response = await call_next(request)
        
        # 2. call_next 返回的是 Response 对象
        # 包含响应的所有信息
        print(f"Response status: {response.status_code}")
        print(f"Response headers: {dict(response.headers)}")
        
        # 3. 可以修改响应体（注意：响应体可能已经流式传输）
        # 简单响应可以直接修改
        if isinstance(response, JSONResponse):
            response.body = modified_body
        
        return response
```

## 常用中间件

### 跨域中间件

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://example.com"],  # 允许的源
    allow_origin_regex=None,                 # 正则表达式匹配源
    allow_methods=["GET"],                   # 允许的方法
    allow_headers=["*"],                     # 允许的请求头
    allow_credentials=False,                 # 是否允许凭证
    expose_headers=[],                       # 暴露的响应头
    max_age=600,                            # 预检请求缓存时间
)
```

### Gzip中间件

#### 常用参数

```python
app.add_middleware(
    GZipMiddleware,
    minimum_size=500,      # 最小压缩大小（字节）
    compresslevel=9,       # 压缩级别（1-9，9最高压缩率）
)
```

#### 示例

```python
app.add_middleware(GZipMiddleware, minimum_size=1000)
```

### https重定向中间件

```python
app.add_middleware(HTTPSRedirectMiddleware)
```

### 信任主机中间件

```python
app.add_middleware(
    TrustedHostMiddleware,
    allowed_hosts=["example.com", "*.example.com"]
)
```
