使用pydantic模型类构造字典

# 构造模型

  1. 由于我们并不知道一些数据，是否满足我们的逻辑要求 我们可以构造一个pydantic 的数据类，此处是TestClass
  2. 我们在类里面，通过Field，来确定每个字段的合法值
  3. 在转换时，通过TestClase(** 转换的数据)，来进行构造
  4. 如果数据不符合，就会报错，如果符合
from pydantic import BaseModel, Field

class TestClase(BaseModel):
    
    name: str = Field(..., description="名字")
    age: int = Field(..., description="年龄", ge=0, le=150)

result = TestClase(**{"name": "张三", "age": -1})
print(result)

# 额外参数接收

对于一些需要卡控的参数 或者后续逻辑用不到，但是仍需要在数据流出现的参数，我们需要接收，可以用如下方法

# 关键代码

  1. V1版本写法

    class Config:
        extra = "allow"
    1
    2

  2. V2版本写法

    model_config = ConfigDict(extra="allow")
    1

# 完整示例

  1. V1写法

    from pydantic import BaseModel, Field

class TestClass(BaseModel):
    class Config:
        extra = "allow"

    name: str = Field(..., description="名字")
    age: int = Field(..., description="年龄", ge=0, le=150)

result = TestClass(**{"name": "张三", "age": 10, "height": 180})
print(result)
print(result.height)       # 180
print(result.dict())       # 包括 height

    from pydantic import BaseModel, Field, ConfigDict

class TestClass(BaseModel):
    model_config = ConfigDict(extra="allow")
    name: str = Field(..., description="名字")
    age: int = Field(..., description="年龄", ge=0, le=150)

result = TestClass(**{"name": "张三", "age": 10, "height": 180})
print(result)
print(result.height)
