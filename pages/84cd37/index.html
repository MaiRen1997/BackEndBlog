# 同步与异步代码

前端也有async-await笔记，具体可以通过搜索关于async-await的使用,查看文章

里面比较详细的讲述了async-await的作用

# 同步代码

import time

def task(name, duration):
    print(f"{name} 开始执行")
    time.sleep(duration)  # 模拟耗时操作
    print(f"{name} 执行完成")
    return name

def sync_example():
    print("=== 同步执行 ===")
    start = time.time()
    
    task("任务A", 3)  # 等待3秒
    task("任务B", 2)  # 再等待2秒
    task("任务C", 1)  # 再等待1秒
    
    print(f"总耗时: {time.time() - start:.2f}秒")
    # 输出: 总耗时: 6.00秒 (3+2+1)

# 异步代码

import asyncio
import time

async def async_task(name, duration):
    print(f"{name} 开始执行")
    await asyncio.sleep(duration)  # 异步等待，不会阻塞
    print(f"{name} 执行完成")
    return name

async def async_example():
    print("=== 异步执行 ===")
    start = time.time()
    
    # 并发执行三个任务
    tasks = [
        async_task("任务A", 1),
        async_task("任务B", 1),
        async_task("任务C", 1)
    ]
    
    results = await asyncio.gather(*tasks)
    print(f"总耗时: {time.time() - start:.2f}秒")
    # 输出: 总耗时: 3.00秒 (最长的任务时间)

# 运行异步代码
asyncio.run(async_example())
# 任务A 开始执行
# 任务B 开始执行
# 任务C 开始执行
# 任务A 执行完成
# 任务B 执行完成
# 任务C 执行完成

# 说明

  1. python的异步也是单线程
    1. 当运行到耗时较长的任务时，会一直等待，阻塞后续程序执行
    2. 为了避免等待，可以使用异步，在等待某个任务完成时，程序可以切换到其他任务，提高效率
  2. javaScript是单线程
    1. 当运行到耗时较长的任务时，会自动加入到宏任务里面，等待后面的任务执行完，再执行当前任务
    2. 任务队列是：同步代码、微任务、宏任务
    3. 执行顺序：同步代码 → 所有微任务 → 一个宏任务
      1. 微任务：setTimeout, setInterval
      2. 宏任务：Promise, async/await
    4. 使用async、await 关键字，是将该函数体内，耗时较长的任务，以及后续任务一起挂起，等执行完之后，再执行当前任务
